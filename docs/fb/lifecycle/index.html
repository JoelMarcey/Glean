lifecycle

  • The job invokes glean --service glean.write kickoff <args> to create the database. Note that databases are normally created on the glean.write tier; it won't work to create a database on glean.query.prod.

  • At this point the database is in the Incomplete state. Queries are supported in this state, and always reflect the current contents.

  • Facts are written to the database using the methods described in [[Glean/Write]], and finally the database is closed by invoking glean --service glean.write finish <args> or the appropriate Thrift method.

  • The database is now in the Complete state.

  • If backups are allowed for this database, then the database is uploaded to backup storage, currently Manifold.

    • This is controlled by a setting in configerator: DatabaseBackupPolicy.

  • The query servers notice the new DB on Manifold, and download it.

    • This is controlled by configerator: DatbaseRestorePolicy and DatabaseRetentionPolicy.

  • After it is downloaded on one or more query servers, the database will be available for queries via the usual tools and APIs.
